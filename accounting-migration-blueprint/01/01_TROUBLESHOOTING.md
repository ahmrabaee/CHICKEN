# استكشاف الأخطاء — Blueprint 01

## 1. خطأ 404 عند إنشاء قيد (POST journal-entries)

**العراض:** `POST http://localhost:3000/v1/accounting/journal-entries` يعيد `404 Not Found`

**التحقق:**
1. تشغيل الـ Backend: `cd app/backend && npm run start:dev`
2. التأكد أن `GET http://localhost:3000/v1/accounting/accounts` يعيد 200 (قائمة الحسابات)
3. إن كان GET يعمل و POST لا يعمل: المسار صحيح، والمشكلة قد تكون في الـ body أو التحقق

**حلول مقترحة:**
- إعادة تشغيل الـ Backend بعد أي تعديل على الـ controller
- التحقق من أن المستخدم مسجل دخول وحاصل على دور `admin` أو `manager`
- التحقق من الـ Network tab: هل الطلب يصل فعلاً إلى `localhost:3000` أم إلى عنوان آخر (مثلاً بسبب proxy)؟

**المسار الصحيح:** `POST /v1/accounting/journal-entries`  
**الـ Controller:** `AccountingController` — `@Post('journal-entries')`

## 2. الوحدات النقدية (Minor Units) — لماذا × 100؟

### المعنى

- **Minor units** = أصغر وحدة في العملة (قرش/هللة/سنت).
- في الشيكل: 1 شيكل = 100 قرش → 10.00 ₪ = 1000 وحدة صغيرة.

### لماذا تستخدمها التطبيقات؟

| النظام/الشركة | الأسلوب |
|---------------|---------|
| Stripe (مدفوعات) | المبالغ بالـ cents (دولار واحد = 100) |
| Banks | حسابات بالوحدة الصغيرة لتجنب أخطاء التقريب |
| محاسبة احترافية | تخزين أعداد صحيحة بدلاً من float |

### السبب التقني

```javascript
0.1 + 0.2 === 0.3  // false في JavaScript!
// النتيجة: 0.30000000000000004
```

استخدام `float`/`double` للمال يسبب أخطاء تقريب. استخدام أعداد صحيحة (مثلاً 1000 = 10.00) يتجنب ذلك.

### إمكانية تخزين الرقم كما يُدخل

- **نعم** يمكن تخزين 1000 كـ 1000 (شيكل).
- **عيب**: فقدان الكسور (مثل 10.50 أو 123.45).
- إذا كانت كل المبالغ أعداداً صحيحة بالشيكل فقط — يمكن التبديل.
- إذا ستُضاف كسور لاحقاً — يُفضّل الإبقاء على الوحدات الصغيرة.
