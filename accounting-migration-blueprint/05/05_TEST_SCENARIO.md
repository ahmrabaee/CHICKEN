# Blueprint 05 — سيناريو اختبار شامل
## Tax Engine & VAT Architecture (محرك الضرائب ومعمارية ضريبة القيمة المضافة)

**الإصدار:** 1.0  
**التاريخ:** فبراير 2026  
**الاعتماد:** يتطلب Blueprint 01 (دليل الحسابات) و Blueprint 02 (GL Engine) منفّذين

---

## الهدف

اختبار كامل لتعديلات Blueprint 05 (محرك الضرائب ومعمارية VAT) يشمل:
- **تفعيل/تعطيل Tax Engine** من إعدادات المحاسبة
- **اختيار قالب الضريبة** في نقطة البيع (POS)
- **عرض المجاميع** — صافي المبلغ، الضريبة التقديرية، الإجمالي قبل الحفظ
- **صفحة إدارة قوالب الضرائب** — عرض، فلتر، حذف
- **صفحة تقرير VAT** — Output VAT، Input VAT، صافي المستحق، تفصيل حسب الحساب
- **إنشاء بيع مع ضريبة** — إرسال taxTemplateId وحفظ الفاتورة

## 4. اختبار نقطة البيع (POS) — اختيار قالب الضريبة

| الخطوة | الإجراء | النتيجة المتوقعة |
|--------|---------|------------------|
| 4.1 | فعّل Tax Engine من الإعدادات | — |
| 4.2 | من القائمة: **نقطة البيع** أو **POS** | تفتح شاشة نقطة البيع |
| 4.3 | أضف صنفاً أو أكثر إلى السلة | يظهر المجموع الفرعي |
| 4.4 | ابحث عن حقل **قالب الضريبة** أو **Tax Template** | يظهر Select أو قائمة اختيار |
| 4.5 | افتح قائمة الاختيار | خيارات: "بدون ضريبة" + قوالب المبيعات (VAT 15% Sales إلخ) |
| 4.6 | اختر **VAT 15% Sales** (أو أي قالب مبيعات) | يُحدّد القالب |
| 4.7 | تحقق من عرض المجاميع | يظهر: صافي المبلغ، الضريبة (≈15٪)، الإجمالي |
| 4.8 | غيّر الكميات أو أضف أصنافاً | تُحدَّث المجاميع (صافي + ضريبة + إجمالي) |
| 4.9 | اختر **بدون ضريبة** | الضريبة = 0، الإجمالي = الصافي |
| 4.10 | مسح السلة | يُعاد taxTemplateId إلى null |

### 4.11 مثال حساب المجاميع (قالب 15٪)

| صافي المبلغ (₪) | الضريبة التقديرية (15٪) | الإجمالي |
|------------------|-------------------------|----------|
| 100.00 | 15.00 | 115.00 |
| 50.00 | 7.50 | 57.50 |
| 200.00 | 30.00 | 230.00 |

**ملاحظة:** المبالغ في الـ API بصيغة minor units (قرش). ₪100 = 10000.

## 5. اختبار إنشاء بيع مع ضريبة

| الخطوة | الإجراء | النتيجة المتوقعة |
|--------|---------|------------------|
| 5.1 | في POS: أضف أصنافاً، اختر **VAT 15% Sales** | — |

| 5.5 | اذهب إلى **المحاسبة** → **قيود اليومية** | — |

**API للتحقق:**

## 6. اختبار تقرير ضريبة القيمة المضافة (VAT Report)

| الخطوة | الإجراء | النتيجة المتوقعة |
|--------|---------|------------------|
| 6.1 | من القائمة: **التقارير** | تفتح صفحة التقارير |
| 6.2 | ابحث عن رابط **ضريبة القيمة المضافة** أو **VAT** | يظهر رابط |
| 6.3 | اضغط على رابط تقرير VAT | يفتح `/reports/vat` |
| 6.4 | تحقق من فلتر الفترة | حقول: تاريخ البداية، تاريخ النهاية |
| 6.5 | حدّد فترة (مثلاً الشهر الحالي) | يُقبل |
| 6.6 | تحقق من عرض الملخص | بطاقات: Output VAT، Input VAT، صافي المستحق |
| 6.7 | تحقق من جدول التفصيل | أعمدة: الحساب، Output، Input |
| 6.8 | إن لم تكن هناك مبيعات/مشتريات بالضريبة | Output VAT = 0 أو صغير، Input VAT = 0 أو صغير |
| 6.9 | بعد إنشاء بيع مع ضريبة (الخطوة 5) | Output VAT يُحدَّث حسب الفترة |
| 6.10 | غيّر الفترة | تُحدَّث الأرقام |

**API للتحقق:**
```
GET /v1/tax/vat-report?startDate=2026-02-01&endDate=2026-02-28
Response: {
  outputVat: number,
  inputVat: number,
  netVatPayable: number,
  byAccount: [{ accountId, accountCode, accountName, output, input }],
  byRate: [{ rate, output, input }]
}
```

## 9. ترتيب التنفيذ الموصى به

```
1. تسجيل الدخول (admin / Admin@123)
2. الإعدادات → المحاسبة → تفعيل Tax Engine
3. الإعدادات → قوالب الضرائب: عرض، فلتر، حذف (قالب غير مستخدم)
4. نقطة البيع: إضافة أصناف، اختيار قالب ضريبة (VAT 15% Sales)
5. التحقق من عرض صافي + ضريبة + إجمالي
6. إتمام البيع مع الضريبة
7. المحاسبة → قيود اليومية: التحقق من قيد البيع (إيراد + VAT Payable)
8. التقارير → ضريبة القيمة المضافة: فلتر الفترة، عرض الملخص والتفصيل
9. API: GET /tax/templates، GET /tax/vat-report
10. (اختياري) حذف قالب غير مستخدم عبر UI أو API
```

## 11. استكشاف الأخطاء

| المشكلة | الحل |

| قوالب الضرائب فارغة | تأكد من `db:seed`؛ تحقق من `GET /v1/tax/templates` |

## 12. ملاحظات إضافية

- **الوحدات:** المبالغ في الـ API بصيغة **minor units** (قرش). ₪10.00 = 1000.
- **النسب:** في قوالب الضرائب تُخزّن basis points. 15٪ = 1500.
- **الـ seed:** يضيف قالب "VAT 15% Sales" (مبيعات) و"VAT 15% Purchases" (مشتريات).
- **إنشاء/تعديل قالب:** حوار TaxTemplateFormDialog قد لا يكون منفّذاً؛ العرض والحذف متوفران.
- **Purchase مع ضريبة:** Phase 6 اختياري؛ قد لا يكون مشمولاً في الواجهة الحالية.
