# خطة شاملة: نظام الباركود المخصص (Custom Barcode System)

## 1. نظرة عامة

### 1.0 متطلب ثابت: 25 رقم
**الباركود النهائي يجب أن يكون بالضبط 25 رقم.**

**توزيع مقترح (25 رقم):**

| المقطع | الطول | المثال | الوصف |
|--------|-------|--------|-------|
| رمز البداية | 1 | 2 | باركود داخلي (وزن/سعر) |
| كود الصنف | 6 | 100001 | PLU أو كود المنتج |
| الوزن (غرام) | 5 | 01250 | 1.250 كغ = 1250 غرام |
| السعر الإجمالي مع الضريبة | 7 | 0002500 | 25.00 وحدة نقدية |
| السعر | 5 | 02250 | 22.50 وحدة نقدية |
| خانة تحقق | 1 | X | حساب تلقائي |

**مثال كامل:** `2100001012500002500022507` = 25 رقم
- 2 = بداية
- 100001 = كود الصنف
- 01250 = 1.250 كغ
- 0002500 = السعر مع الضريبة
- 02250 = السعر
- 7 = خانة تحقق

**التحقق:** النظام يرفض أي باركود طوله ≠ 25 عند تفعيل الباركود المخصص.

---

### 1.1 الهدف
اعتماد نظام باركود مخصص يدعم:
- **باركود متغير الوزن**: يشفّر الوزن والسعر وكود الصنف في الباركود نفسه
- **تكوين مرن**: 8 أنواع بيانات (Type1–Type8) مع تعيين خانة/قيمة لكل نوع
- **استخدام في نقطة البيع**: مسح → تحليل → إضافة للسلة تلقائياً
- **استخدام في المشتريات والمخزون**: بحث وإدخال بالباركود

### 1.2 أنواع البيانات المدعومة (من الصورة المرجعية)

| النوع | الوصف | مثال الاستخدام |
|------|-------|----------------|
| Weight, Dot 0 | خانة الوزن (الآحاد) | 1.234 → 4 |
| Weight, Dot 1 | خانة الوزن (أعشار) | 1.234 → 3 |
| Weight, Dot 2 | خانة الوزن (أعشار مئوية) | 1.234 → 2 |
| Weight, Dot 3 | خانة الوزن (أعشار ألوف) | 1.234 → 1 |
| Barcode start code | رمز بداية الباركود | 2 = باركود داخلي |
| Item Code | كود الصنف | 6 خانات |
| Total Price with Tax | السعر الإجمالي مع الضريبة | 7 خانات |
| Price | السعر | 7 خانات |

## 2. الباك إند (Backend)

### 2.1 إعدادات الباركود (Settings)

**إضافة في `seed.ts` – مجموعة `barcode`:**

```ts
// Custom Barcode Format Configuration
{ key: 'barcode.custom_enabled', value: 'true', description: 'تفعيل الباركود المخصص', dataType: 'boolean', settingGroup: 'barcode' },
{ key: 'barcode.type1', value: 'weight_dot_0', description: 'النوع 1', dataType: 'string', settingGroup: 'barcode' },
{ key: 'barcode.type1_digit', value: '1', description: 'خانة النوع 1', dataType: 'string', settingGroup: 'barcode' },
{ key: 'barcode.type2', value: 'weight_dot_1', description: 'النوع 2', dataType: 'string', settingGroup: 'barcode' },
{ key: 'barcode.type2_digit', value: '2', description: 'خانة النوع 2', dataType: 'string', settingGroup: 'barcode' },
// ... type3–type8 بنفس النمط
{ key: 'barcode.start_code', value: '2', description: 'رمز بداية الباركود الداخلي', dataType: 'string', settingGroup: 'barcode' },
```

**قيم النوع المسموحة:**
- `weight_dot_0`, `weight_dot_1`, `weight_dot_2`, `weight_dot_3`
- `barcode_start_code`
- `item_code`
- `total_price_with_tax`
- `price`

**قيم Digit/Value:** رقم خانة (1–N) أو قيمة ثابتة

**التحقق من الطول:** مجموع أطوال جميع الأنواع (Type1–Type8) يجب أن يساوي **25** بالضبط. واجهة الإعدادات تعرض التحقق وتنبه المستخدم إن كان المجموع ≠ 25.

### 2.2 خدمة تحليل الباركود (Barcode Parser Service)

**ملف جديد:** `app/backend/src/barcode/barcode.service.ts`

```ts
interface BarcodeParseResult {
  itemCode: string;           // كود الصنف المستخرج
  weightKg?: number;         // الوزن بالكيلو (إن وُجد)
  price?: number;            // السعر (إن وُجد، minor units)
  totalPriceWithTax?: number; // السعر مع الضريبة (إن وُجد)
  rawBarcode: string;
}
```

**الوظائف:**
- `parseCustomBarcode(barcode: string, config: BarcodeConfig): BarcodeParseResult`
- `extractItemCode(barcode: string): string` — لاستخدام البحث بالكود فقط
- قراءة الإعدادات من `SettingsService.getByGroup('barcode')`

### 2.3 واجهة API جديدة

**ملف:** `app/backend/src/barcode/barcode.controller.ts`

| Method | Endpoint | الوصف |
|--------|----------|-------|
| GET | `/v1/barcode/parse?barcode=xxx` | تحليل الباركود وإرجاع النتيجة |
| GET | `/v1/barcode/config` | إرجاع إعدادات الباركود الحالية |
| PUT | `/v1/barcode/config` | تحديث إعدادات الباركود |

### 2.4 تعديل Items Service

**تعديل `findByBarcode` و/أو إضافة:**

- `findByBarcodeOrParsed(barcode: string)`:
  1. إن كان الباركود المخصص مفعّل: استدعاء `BarcodeService.parseCustomBarcode`
  2. استخراج `itemCode` من النتيجة
  3. البحث عن صنف بـ `code === itemCode` أو `barcode === barcode` (للتوافق مع الباركود الثابت)
  4. إرجاع الصنف مع الوزن والسعر المستخرجين (إن وُجدت) للاستخدام في السلة

**أو:** الإبقاء على `findByBarcode` للبحث المباشر، وإضافة:
- `POST /v1/items/lookup` — يقبل `{ barcode: string }` ويرجع `{ item, weightKg?, price? }` بعد التحليل

## 3. الفرونت إند (Frontend)

### 3.1 صفحة إعدادات الباركود المخصص

**المسار:** الإعدادات → تبويب جديد "الباركود" أو ضمن "نقطة البيع"

**المكونات:**
- `BarcodeSettingsTab.tsx` — تبويب كامل في `Settings.tsx`
- جدول تكوين (Type1–Type8):
  - عمود: نوع البيانات (Select)
  - عمود: Digit/Value (Input رقمي)
- سطر ملخص في الأسفل (مثل الصورة المرجعية)
- زر "حفظ"
- زر "اختبار الباركود" — حقل إدخال + عرض نتيجة التحليل

### 3.2 نقطة البيع (POS)

**تعديلات `POS.tsx`:**

1. **تفعيل زر "مسح باركود":**
   - إزالة `disabled`
   - عند النقر: فتح Dialog صغير بحقل إدخال (للإدخال اليدوي أو لوحة المفاتيح من الماسح)
   - أو: حقل إدخال مخفي يركز عليه تلقائياً عند النقر، يستقبل من الماسح (ينتهي بـ Enter)

2. **معالج الباركود:**
   - عند استلام باركود (Enter أو زر تحليل):
     - استدعاء `GET /v1/barcode/parse?barcode=xxx` أو `POST /v1/items/lookup`
     - إن وُجد صنف:
       - إضافة للسلة بالوزن المستخرج (إن وُجد) أو 1 كغ افتراضي
       - استخدام السعر المستخرج (إن وُجد) بدل `defaultSalePrice`
     - إن لم يُوجد: عرض toast "الباركود غير معروف"

3. **دعم الماسح الضوئي:**
   - `useEffect` مع `keydown`: تجميع الأحرف حتى Enter
   - إن كانت السرعة سريعة (ماسح) → معالجة كباركود
   - إن كانت بطيئة (إدخال يدوي) → تجاهل أو معالجة كبحث

4. **تحديث placeholder البحث:**
   - "ابحث عن منتج بالاسم أو الكود أو الباركود..."

### 3.3 المشتريات (Purchases)

**صفحة إضافة/تعديل مشتريات:**
- حقل "مسح باركود" أو أيقونة بجانب إضافة صنف
- نفس منطق التحليل والبحث
- إضافة الصنف لسطر الفاتورة بالكمية/الوزن المستخرج

### 3.4 المخزون (Inventory)

**صفحة المخزون / جرد المخزون:**
- حقل مسح باركود للبحث السريع عن صنف
- عرض تفاصيل الصنف والكمية الحالية

### 3.5 ملفات الخدمات (Services)

**جديد:** `barcode.service.ts`
```ts
parseBarcode(barcode: string): Promise<BarcodeParseResult>

```

**تعديل:** `item.service.ts`
```ts

```

## 6. ملاحظات تقنية

### 6.1 ترتيب الـ Routes في Items Controller
- `GET /items/barcode/:barcode` يجب أن يسبق `GET /items/:id` لتجنب تفسير "barcode" كـ id.

### 6.2 تنسيق الباركود
- الطول والمواضع تعتمد على إعدادات المستخدم
- التحليل يعتمد على التكوين المحفوظ فقط

### 6.3 التوافق مع الباركود الثابت
- إن لم يكن الباركود المخصص مفعّلاً، أو فشل التحليل: البحث المباشر بـ `barcode` في جدول Items

### 6.4 التحقق من 25 رقم
- عند تفعيل الباركود المخصص: أي باركود طوله ≠ 25 يُرفض مع رسالة واضحة
- الباركود الثابت (EAN-13، UPC-A، إلخ) يعمل خارج وضع الباركود المخصص (بحث مباشر)
