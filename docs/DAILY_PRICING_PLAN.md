# خطة التسعير اليومي (Daily Pricing Plan)

> ✅ **تم التنفيذ** — المراجعة لهذه الوثيقة بعد التطبيق.

## الهدف

تمكين تحديد أسعار البيع **يومياً** بدل الاعتماد على السعر الافتراضي الثابت فقط. المنطق:
- كل يوم السعر قد يرتفع شيكل أو ينقص
- عند فتح صفحة البيع، تُعرض خيارات تسعير اليوم
- إمكانية نسخ تسعيرة الأمس بضغطة زر للراحة

---

## الوضع الحالي في النظام

### كيف يعمل السعر حالياً

| المكون | التفاصيل |
|--------|----------|
| **Item** | يحتوي على `defaultSalePrice` (ثابت، وحدات فرعية لكل كجم) |
| **POS** (`/sales/new`) | يستخدم `item.defaultSalePrice` مباشرة عند إضافة المنتج للسلة (سطر 95 في POS.tsx) |
| **منافذ البيع** | نقطة البيع فقط: `/sales/new` (نفس صفحة POS) |
| **إنشاء الفاتورة** | يُرسل `pricePerKg` من السلة إلى API — أي سعر يُعرض في السلة يُخزَّن مع البيع |

### الاستنتاج

- **لا يوجد تسعير يومي حالياً** — السعر يأتي دائماً من `Item.defaultSalePrice`
- **تعديل الباك اند ضروري** — لا يمكن تنفيذ "استخدام تسعيرة الأمس" بدون تخزين تاريخي للأسعار
- **تعديل الفرونت اند ضروري** — لعرض وضبط أسعار اليوم قبل/أثناء البيع

## البنية المقترحة

### 1. نموذج البيانات (Backend)

**جدول جديد: `ItemDailyPrice`**

```prisma
model ItemDailyPrice {
  id           Int      @id @default(autoincrement())
  itemId       Int      @map("item_id")
  pricingDate  DateTime @map("pricing_date") // YYYY-MM-DD (بدون وقت)
  pricePerKg   Int      @map("price_per_kg") // وحدات فرعية لكل كجم
  branchId     Int?     @map("branch_id")    // null = جميع الفروع (للمرحلة الأولى)
  createdAt    DateTime @default(now()) @map("created_at")
  createdById  Int?     @map("created_by_id")

  item   Item    @relation(...)
  branch Branch? @relation(...)
  createdBy User? @relation(...)

  @@unique([itemId, pricingDate, branchId])
  @@index([pricingDate])
  @@map("item_daily_prices")
}
```

- **المرحلة الأولى:** `branchId` = null → تسعير موحد لكل الفروع
- **لاحقاً:** إمكانية دعم تسعير لكل فرع

### 2. واجهات الـ API (Backend)

| Method | Endpoint | الوصف |
|--------|----------|-------|
| `GET` | `/daily-pricing?date=YYYY-MM-DD` | جلب أسعار يوم معيّن. يُرجع `{ items: [{ itemId, pricePerKg, itemName }] }` |
| `GET` | `/daily-pricing/yesterday` | اختصار لجلب أسعار أمس |
| `POST` | `/daily-pricing` | حفظ أسعار يوم: `{ date, prices: [{ itemId, pricePerKg }] }` |
| `POST` | `/daily-pricing/copy-from-yesterday` | نسخ أسعار أمس إلى اليوم: `{ date }` |

### 3. منطق اختيار السعر

```
عند إضافة منتج للسلة في POS:
  if (يوجد سعر يومي لليوم الحالي لهذا المنتج)
    استخدم سعر اليوم
  else
    استخدم item.defaultSalePrice (كما هو حالياً)
```

- إذا لم تُضبط أسعار اليوم → النظام يعمل كما هو الآن بالسعر الافتراضي
- إذا وُجدت أسعار يومية → تُستخدم بدل `defaultSalePrice`

## وضع واجهة التسعير اليومي

### الخيارات المدروسة

| الخيار | الوصف | مزايا | عيوب |
|--------|--------|-------|------|
| **أ** | صفحة مستقلة `/pricing/daily` | وضوح، إدارة مركزية | خطوة إضافية قبل البيع |
| **ب** | نافذة منبثقة عند أول دخول لـ POS اليوم | مدمجة في تدفق البيع | قد تُستدعى أكثر من مرة |
| **ج** | بانر/قسم علوي في POS قابل للطي | مرئية، تعديل سريع | يشغل مساحة |
| **د** | شاشة وسيطة قبل POS عند غياب التسعير | تدفق واضح (تعيين → بيع) | خطوة إضافية |

### التوصية: الخيار **د** مع تحسين

**تدفق مقترح:**

1. المستخدم يفتح `/sales/new` (POS)
2. التطبيق يتحقق: هل هناك أسعار يومية لليوم الحالي؟
3. **إن لم توجد:**
   - عرض شاشة "تسعير اليوم" قبل دخول POS:
     - جدول: المنتج | السعر الحالي (من أمس أو default) | حقل تعديل السعر
     - زر **"استعمال تسعيرة الأمس"** — ينسخ أمس ويحفظ
     - زر **"استخدام الأسعار الافتراضية"** — يستخدم defaultSalePrice لكل منتج (بدون تخزين يومي)
     - زر **"متابعة للبيع"** — يحفظ التعديلات وينتقل لـ POS
4. **إن وُجدت أسعار اليوم:**
   - الدخول مباشرة إلى POS
   - زر صغير (مثل أيقونة) "تعديل أسعار اليوم" لفتح نفس الشاشة عند الحاجة

## حالات خاصة

| الحالة | السلوك |
|--------|--------|
| أول يوم للنظام | لا أسعار أمس → الزر "استعمال تسعيرة الأمس" معطّل أو ينسخ من defaultSalePrice |
| منتج جديد أُضيف اليوم | يظهر في جدول التسعير بـ defaultSalePrice؛ يُمكن تعديله |
| دخول POS دون إعداد أسعار اليوم | استخدام defaultSalePrice لكل المنتجات (سلوك حالي) |
| تعديل السعر أثناء وجود سلّة | السعر الموجود في السلة لا يتغيّر تلقائياً؛ البيع بالسعر عند الإضافة |
