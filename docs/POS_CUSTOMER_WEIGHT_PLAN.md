## تحليل المشكلة الحالية

### 1. خلل منطق إلزام الزبون

**الكود الحالي (السطر ≈ 258):**
```ts
const needsCustomer =
  saleType === "credit" ||
  (saleType === "cash" && paidMinor > 0 && paidMinor < totalMinor - 1);
```

**الخلل المكتشف:**
| الحالة | القيمة | `needsCustomer` الحالي | الصحيح |
|--------|--------|------------------------|--------|
| نقدي + مبلغ مدفوع فارغ (لم يُدخَل) | `paidAmount = ""` → `paidMinor = 0` | ❌ `false` | ✅ `false` (يُعامَل كدفع كامل) |
| نقدي + كتب "0" صراحةً | `paidAmount = "0"` → `paidMinor = 0` | ❌ `false` (لا يطلب زبون!) | ✅ `true` (دين كامل) |
| نقدي + دفع جزئي | `paidMinor = 500, total = 2000` | ✅ `true` | ✅ `true` |
| نقدي + دفع كامل | `paidMinor = total` | ✅ `false` | ✅ `false` |
| آجل | أي قيمة | ✅ `true` | ✅ `true` |

**جذر المشكلة:** `toMinor("")` و `toMinor("0")` كلاهما يُعيدان `0`، فلا يمكن التفريق بين "مبلغ غير مُدخَل" و"صفر مُدخَل صراحةً".

---

## القواعد الصحيحة (Logic Rules)

```
يُلزَم بإدخال زبون إذا:
  ├── saleType === "credit"          ← دائماً
  └── saleType === "cash" AND:
        ├── paidAmount !== ""         ← المستخدم أدخل رقماً
        └── paidMinor < totalMinor - 1 ← والمبلغ أقل من الإجمالي
            (يشمل: "0" = دين كامل، أو جزئي)
```

## الإصلاح المطلوب: منطق إلزام الزبون

### الملف: `app/frontend/src/pages/POS.tsx`

**الخطوة 1 — تصحيح `needsCustomer`:**

```ts
// القديم (خطأ):
const needsCustomer =
  saleType === "credit" ||
  (saleType === "cash" && paidMinor > 0 && paidMinor < totalMinor - 1);

// الجديد (صحيح):
const isExplicitUnderPayment =
  paidAmount !== "" && paidMinor < totalMinor - 1;
const needsCustomer =
  saleType === "credit" || (saleType === "cash" && isExplicitUnderPayment);
```

**الخطوة 2 — تحديث رسائل التحذير:**

| الحالة | الرسالة الحالية | الرسالة الجديدة |
|--------|-----------------|-----------------|
| `credit` | "البيع الآجل يتطلب بيانات الزبون..." | ✅ نفسها |
| `cash` + `paidAmount = "0"` | لا تظهر رسالة ❌ | ⚠️ "دفع صفر = دين كامل، أدخل بيانات الزبون" |
| `cash` + جزئي | "الدفع الجزئي يتطلب..." | ✅ نفسها |

**الخطوة 3 — منع إتمام البيع:**

في `handleCompleteSale` نضيف guard صريح قبل إرسال الـ DTO:
```ts
if (needsCustomer && !hasCustomerInfo) {
  // toast.error / يُركّز على حقل الزبون
  return;
}
```
(حالياً `canCompleteSale` يمنع الزر، لكن لا guard داخلي إذا استُدعيت الدالة بطريقة أخرى)

## تطوير عداد الوزن

### المشكلة الحالية
| المشكلة | التفاصيل |
|---------|----------|
| ±0.5 ثابتة | لا توجد خطوات مرنة (±0.25، ±1) |
| لا numpad | الإدخال يدوي فقط، غير ملائم للـ POS اللمسي |
| الضغط على البطاقة | يضيف 1 كجم دائماً دون سؤال |
| لا تأكيد بصري | المستخدم لا يرى الوزن واضحاً قبل الإضافة |
| غرامات / كيلو | لا يعرض بالوحدتين |

### مكونات جديدة مطلوبة

#### `WeightEntryDialog.tsx`
**المسار:** `app/frontend/src/components/pos/WeightEntryDialog.tsx`

```tsx
interface WeightEntryDialogProps {
  item: Item | null;           // null = مغلق
  onConfirm: (item: Item, quantityKg: number) => void;
  onClose: () => void;
}
```

**الخصائص:**
- Numpad رقمي (0–9، نقطة، حذف) — أزرار كبيرة قابلة للضغط باللمس
- شاشة عرض تظهر: وزن بالكجم + غرام + المجموع
- أزرار خطوات سريعة: **+0.25** | **+0.5** | **+1** | **-0.5**
- أزرار preset: `250g`, `500g`, `750g`, `1kg`, `1.5kg`, `2kg`
- تحقق: لا يقبل 0 أو قيمة فارغة
- يُغلق بـ Escape أو الضغط خارجه

#### تعديلات على `POS.tsx`
- استبدال `onClick={() => addToCart(item)}` بـ `onClick={() => setWeightDialogItem(item)}`
- إضافة `state: weightDialogItem: Item | null`
- `onConfirm: (item, qty) => { addToCart(item, qty); setWeightDialogItem(null); }`

#### عداد في السلة (Cart Row)
تعديل الأزرار في جدول السلة:

```
الحالي:  [−0.5] [1.50] [+0.5]
الجديد:  [−0.25] [−0.5] [قيمة قابلة للضغط → يفتح numpad] [+0.5] [+0.25]
```

- النقر على الرقم في السلة يفتح `WeightEntryDialog` بالوزن الحالي
- أزرار ±0.25 و ±0.5 تبقى للتعديل السريع
- عرض `2.50 كجم` (بدلاً من `2.5`)
