# خطة إصلاح صفحة تحوويل المخزون (Stock Transfer)

## 1. الفهم الحالي vs المطلوب

### 1.1 الوضع الحالي
- **المصدر**: دجاج خام فقط (CHK-RAW-01) — من دفعات الشراء
- **الوجهة**: أي منتج نهائي (ما عدا الدجاج الخام) — قيد خاطئ
- **التدفق**: اختيار دفعة دجاج خام → توزيع الكيلوهات على عدة منتجات
- **القيود**: حركات مخزون صحيحة (نقص المصدر، زيادة الوجهة)

### 1.2 المطلوب
- **المصدر**: أي منتج له مخزون متاح — بما فيه الدجاج الخام
- **الوجهة**: أي منتج — بما فيه الدجاج الخام
- **التدفق**: اختيار المنتج المصدر → تحديد الكيلوهات → اختيار المنتج الوجهة
- **الحركات**: نقص مخزون المصدر، زيادة مخزون الوجهة

### 1.3 التعريف
> تحويل المخزون = نقل X كجم من منتج (مصدر) إلى منتج آخر (وجهة).  
> **كل المنتجات** — بما فيها الدجاج الخام — يمكن أن تكون مصدرًا أو وجهة.

## 2. الهيكل الحالي (Schema)

```
StockTransfer
├── sourceLotId      → InventoryLot (دفعة واحدة كمصدر)
├── totalWeightGrams
├── totalCostValue
├── lines[]          → StockTransferLine[]
│   ├── itemId       → المنتج الوجهة
│   ├── weightGrams
│   └── unitCost
```

- **InventoryLot**: لكل منتج دفعات (lots) من الشراء أو التحويلات السابقة
- **Inventory**: رصيد إجمالي لكل منتج
- **StockMovement**: حركة مخزون (دخول/خروج)
- **StockLedgerEntry**: قيد محاسبي للمخزون

## 3. التعديلات المطلوبة

### 3.1 Backend

#### أ) تعميم `getAvailableSourceLots`
- **الحالي**: فقط دفعات الدجاج الخام (`itemId = rawChicken`)
- **المطلوب**: دفعات لكل المنتجات التي لها `remainingQuantityGrams > 0`

**الملف**: `stock-transfer.service.ts`

```typescript
// إزالة: RAW_CHICKEN_CODE, CHICKEN_RAW
// إضافة: itemId optional - إذا وُجد نفلتر حسب المنتج
async getAvailableSourceLots(branchId?: number | null, itemId?: number | null)
```

**المنطق الجديد**:
```
where: {
  remainingQuantityGrams: { gt: 0 },
  // إزالة stockTransferId: null — نسمح بكل الدفعات (شراء أو تحويل سابق)
  ...(itemId ? { itemId } : {}),
  ...(branchId != null ? { branchId } : {}),
}
```

**ترتيب**: حسب `receivedAt` (FIFO)

#### ب) تعميم `getTransferrableProducts`
- **الحالي**: كل المنتجات ما عدا الدجاج الخام (قيد خاطئ)
- **المطلوب**: كل المنتجات — بما فيها الدجاج الخام. يمكن استثناء المنتج المصدر فقط (اختياري)

**المنطق**: عرض كل المنتجات النشطة. الدجاج الخام وغيره يمكن أن يكونوا وجهة تحويل.

#### ج) `create` — بدون تغيير
- المنطق الحالي: يقلل المصدر، يزيد الوجهة، ينشئ SLE و StockMovement
- يبقى كما هو لكن بدون التحقق من نوع المصدر (دجاج خام فقط)

#### د) `expiryDate` — اختياري
- **الحالي**: مطلوب
- **المطلوب**: اختياري — إذا لم يُدخل، نستخدم صلاحية دفعة المصدر أو قيمة افتراضية

### 3.2 Frontend

#### أ) صفحة الإنشاء (`StockTransferProfile.tsx`)

**التدفق الحالي**:
1. اختيار دفعة مصدر (دجاج خام فقط)
2. تاريخ صلاحية
3. توزيع على منتجات

**التدفق الجديد**:
1. **اختيار المنتج المصدر** (اختياري)
   - Combobox: منتجات لها مخزون
   - أو عرض كل الدفعات المتاحة مع اسم المنتج
2. **اختيار الدفعة** (إذا كان المنتج له عدة دفعات)
   - أو عرض كل الدفعات المتاحة: `[منتج] — [دفعة] — X كجم`
3. **التوزيع على المنتجات الوجهة**
   - منتج وجهة + كيلوهات (كما هو)
4. **تاريخ الصلاحية** (اختياري)

**الواجهة المقترحة**:
```
┌─────────────────────────────────────────────────────────┐
│ 1. المنتج المصدر                                        │
│    [Combobox: منتج أ — دفعة #123 — 50 كجم متاح]         │
│    أو: [اختر المنتج] → [اختر الدفعة]                   │
├─────────────────────────────────────────────────────────┤
│ 2. التوزيع على المنتجات الوجهة                          │
│    [منتج ب] [كجم] [تكلفة/كجم] [حذف]                    │
│    [منتج ج] [كجم] [تكلفة/كجم] [حذف]                    │
│    [+ إضافة سطر]                                        │
├─────────────────────────────────────────────────────────┤
│ 3. تاريخ الصلاحية (اختياري)                             │
│ 4. ملاحظات                                              │
└─────────────────────────────────────────────────────────┘
```

#### ب) قائمة التحويلات (`StockTransfer.tsx`)
- إضافة عمود: **المنتج المصدر**
- عرض اسم المنتج المصدر بدل "رقم الدفعة" فقط

## 4. الحركات المطلوبة (Inventory)

عند تنفيذ التحويل:

| الحركة | العنصر | التأثير |
|--------|--------|---------|
| **نقص المصدر** | InventoryLot (sourceLotId) | `remainingQuantityGrams -= totalWeightGrams` |
| | Inventory (منتج المصدر) | `currentQuantityGrams -= totalWeightGrams` |
| | StockMovement | `quantityGrams: -totalWeightGrams` |
| | StockLedgerEntry | `qtyChange: -totalWeightGrams` |
| **زيادة الوجهة** | InventoryLot (جديد لكل سطر) | إنشاء دفعة جديدة |
| | Inventory (منتج الوجهة) | `currentQuantityGrams += weightGrams` |
| | StockMovement | `quantityGrams: +weightGrams` |
| | StockLedgerEntry | `qtyChange: +weightGrams` |

المنطق الحالي في `create` يغطي هذا — لا حاجة لتعديل كبير.

## 6. ملاحظات تقنية

### 6.1 الدفعات (Lots)
- المنتجات من الشراء لها دفعات (`purchaseId` / `purchaseLineId`)
- المنتجات من التحويل لها دفعات (`stockTransferId`)
- **قرار**: إزالة شرط `stockTransferId: null` — نعرض كل الدفعات التي لها `remainingQuantityGrams > 0`، سواء من شراء أو تحويل سابق. هذا يسمح بالتحويل من أي منتج له مخزون.

### 6.2 التكلفة
- تكلفة الوجهة: حالياً من `unitCost` في كل سطر
- يمكن اقتراح: تكلفة الوجهة = تكلفة المصدر (نفس التكلفة للكيلو) أو إدخال يدوي

## 7. ملخص الفرق

| العنصر | الحالي | بعد التعديل |
|--------|--------|-------------|
| المصدر | دجاج خام فقط | أي منتج له مخزون (بما فيه الدجاج الخام) |
| الوجهة | أي منتج ما عدا الدجاج الخام | أي منتج (بما فيه الدجاج الخام) |
| الدفعات | دفعات الدجاج الخام فقط | دفعات لكل المنتجات |
| تاريخ الصلاحية | مطلوب | اختياري |
| واجهة المصدر | "الشحنة المصدر (دجاج خام)" | "المنتج المصدر" + "الدفعة" |
| قائمة التحويلات | لا يعرض المنتج المصدر | عمود "المنتج المصدر" |
